CODING AGENT SESSION
====================

Upload file: coding_session_export.txt

Description for the upload:
---------------------------

I built a working AI claims adjudication system with an LLM agent loop, 3-tier OCR, RAG, and 30 test cases. Then I asked Claude Code one question: "Is this truly agentic? Be brutally honest."

Claude Code read through my agent.ts line by line and identified the architectural flaw — lines 398-435 where a deterministic post-processing step overrode the LLM's decisions. If you can't trust the agent and must correct it with if-else, the agent isn't deciding — the rule engine is. The LLM was a "glorified function sequencer."

The session then became an architectural redesign: splitting the system into hard gates (deterministic code for arithmetic, date math, null checks) and judgment calls (exclusion interpretation, cross-document consistency, clinical coherence — things only an LLM can reason about). Includes new tool specs with real input/output schemas, a redesigned agent flow, and a spec-driven development plan.

It shows how I use AI coding tools — not to generate boilerplate, but as a technical peer that reads my actual code and tells me when my architecture is wrong.
